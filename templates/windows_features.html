Paso 3 — Guía de diagnóstico

Referencia
Causas comunes de error de acceso
  • Servicio LanmanWorkstation detenido o desactivado
  • Reglas de firewall de Detección de redes deshabilitadas
  • TCP 445 bloqueado por firewall intermedio
  • El servidor no está activo o no responde en la red
  • Credenciales caducadas o cuenta bloqueada en el servidor
  • Restricciones de directiva de grupo (GPO)
Pasos de verificación recomendados
  1. Confirmar que LanmanWorkstation está en ejecución (Paso 1)
  2. Probar TCP 445 al servidor objetivo (Paso 2)
  3. Verificar resolución de nombre DNS con ping servidor
  4. Revisar firewall local y remoto para puerto 445
  5. Comprobar permisos NTFS y permisos de recurso compartido
  6. Verificar en el servidor: Get-SmbShare / net share
SMB versiones — advertencias
SMB1 está obsoleto y es inseguro (EternalBlue/WannaCry).
Si el Paso 1 muestra SMB1 activo, se recomienda desactivarlo desde Características de Windows o via PowerShell:
Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force
La desactivación de SMB1 no está disponible en esta versión de CleanCPU — úsala solo desde una sesión de PowerShell elevada si el diagnóstico lo requiere.
Limitaciones de este diagnóstico
  • La prueba TCP 445 verifica conectividad de red — no valida credenciales
  • Las conexiones SMB muestran lo que este equipo tiene abierto, no lo que el servidor ve
  • GPO puede bloquear el acceso aunque los servicios estén activos
  • Rutas con caracteres Unicode o especiales no son admitidas por esta herramienta
  • Abrir en Explorador no garantiza éxito si fallan las credenciales

⚠ SMB1/CIFS — Protocolo legado inseguro

Riesgo alto
ADVERTENCIA DE SEGURIDAD
SMB1 es un protocolo con más de 30 años de antigüedad que contiene vulnerabilidades críticas conocidas, incluyendo EternalBlue (CVE-2017-0144) — el exploit utilizado por WannaCry y NotPetya.
Microsoft recomienda no habilitar SMB1 bajo ninguna circunstancia. Habilitarlo solo es justificable para acceder temporalmente a recursos de red extremadamente legados que no pueden actualizarse y que no tienen alternativa viable.
Cargando estado…
Requiere modo de operación EXPERT y privilegios de administrador. Se requiere reinicio para activar. Deshabilite SMB1 en cuanto ya no sea necesario.

🔒 Acceso de invitado inseguro (solo compatibilidad legada)

DESTRUCTIVO
Esta NO es la corrección principal para incompatibilidad de firma SMB.
El acceso de invitado inseguro permite conexiones sin autenticación, lo que representa un riesgo de seguridad significativo. Úselo ÚNICAMENTE si el diagnóstico confirma que el servidor requiere acceso de invitado sin autenticación y no existe alternativa viable. La corrección para la firma SMB es la remediación anterior.
Requiere modo EXPERT y privilegios de administrador. Modifica el registro y reinicia LanmanWorkstation. Deshabilitar en cuanto ya no sea necesario.
